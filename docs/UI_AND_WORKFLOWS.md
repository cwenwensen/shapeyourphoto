# UI And Workflows

## 主界面结构

- 顶部：标题、路径、操作按钮、筛选、任务进度
- 左侧：缩略图结果列表
- 右侧上方：HUD + 指标条图
- 右侧下方：诊断说明、属性信息、Console

## 三种导入入口

### 选择单图

- 通过“选择图片”导入
- 适合快速单张分析

### 追加多图

- 多次选择图片或拖拽多张图
- 应追加到当前等待列表

### 目录扫描

- 通过“选择目录”或拖入文件夹导入
- 应显示扫描进度

## 分析流程

1. 读取文件
2. 基础特征统计
3. 指标计算
4. 问题判断
5. 推荐生成
6. 完成并刷新 UI

## 修复流程

1. 选择“修复当前”或“批量修复勾选”
2. 在修复弹窗中选择自动或手动方案
3. 执行修复
4. 输出到目标目录或覆盖原文件
5. 保持 EXIF、ICC、DPI、XMP 等可保留信息

### 调试模式：修复后选择打开前后对比

- 主界面提供一个会话级复选框
- 默认关闭
- 开启后，仅在本轮修复成功后弹出调试打开窗口
- 窗口只列出本轮成功输出的图片，不混入历史结果
- 用户可勾选要打开的条目，然后批量打开原图和修复图
- 该功能不改变右侧预览区，不内置图片查看器，不改变修复输出逻辑

## 当前已知体验要求

- 右侧不显示大图片预览
- 条图区域尽量高，顶部说明尽量精简
- 弹窗底部按钮必须完整可见
- 条图信息尽量多显示，但不要以可读性为代价
