# UI And Workflows

## 主界面结构

- 顶部：标题、路径、操作按钮、筛选、任务进度
- 左侧：缩略图结果列表
- 右侧上方：HUD + 指标条图
- 右侧下方：诊断说明、属性信息、Console

## 三种导入入口

### 选择单图

- 通过“选择图片”导入
- 适合快速单张分析

### 追加多图

- 多次选择图片或拖拽多张图
- 应追加到当前等待列表

### 目录扫描

- 通过“选择目录”或拖入文件夹导入
- 应显示扫描进度

## 分析流程

1. 读取文件
2. 基础特征统计
3. 指标计算
4. 问题判断
5. 推荐生成
6. 完成并刷新 UI

## 修复流程

1. 选择“修复当前”或“批量修复勾选”
2. 在修复弹窗中选择自动或手动方案
3. 执行修复
4. 输出到目标目录或覆盖原文件
5. 保持 EXIF、ICC、DPI、XMP 等可保留信息

### 调试模式：修复后选择打开前后对比

- 主界面提供一个会话级复选框
- 默认关闭
- 开启后，仅在本轮修复成功后弹出调试打开窗口
- 窗口只列出本轮成功输出的图片，不混入历史结果
- 用户可勾选要打开的条目，然后批量打开原图和修复图
- 该功能不改变右侧预览区，不内置图片查看器，不改变修复输出逻辑

## 当前已知体验要求

- 右侧不显示大图片预览
- 条图区域尽量高，顶部说明尽量精简
- 弹窗底部按钮必须完整可见
- 条图信息尽量多显示，但不要以可读性为代价
## 1.1.3 Maintenance Addendum: Cleanup Candidate Panel

- 主列表之外新增单独的“不适合保留候选”框体。
- 该框体只汇总 analyzer 产出的高风险 cleanup candidates，不会自动勾选任何图片。
- 当本轮分析刚结束且检测到新的 cleanup candidate 时，还会额外弹出一个本轮候选确认对话框，列出缩略图、文件名、主要原因、严重度和置信度。
- 支持：
  - 单个勾选
  - 批量切换所选候选
  - 全选
  - 取消全选
- 左侧主按钮用于执行安全清理：
  - 无勾选时按钮禁用
  - 有勾选后才启用
  - 执行前必须二次确认
- 安全清理顺序：
  - 优先尝试移入系统回收站
  - 若系统不支持，则移动到项目内 `_cleanup_candidates` 隔离目录
