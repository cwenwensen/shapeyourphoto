# UI And Workflows

## 主界面结构

- 顶部：标题、路径、操作按钮、筛选、任务进度
- 左侧：缩略图结果列表
- 右侧上方：HUD + 指标条图
- 右侧下方：诊断说明、属性信息、Console

## 三种导入入口

### 选择单图

- 通过“选择图片”导入
- 适合快速单张分析

### 追加多图

- 多次选择图片或拖拽多张图
- 应追加到当前等待列表

### 目录扫描

- 通过“选择目录”或拖入文件夹导入
- 应显示扫描进度
- 如目录包含子目录，应先弹出扫描范围选择对话框
- 扫描范围选择对话框必须受屏幕高度限制；内容可滚动，底部“取消扫描”不能被小屏或系统缩放遮住
- 默认忽略所有以 `_repair` 开头的目录；用户也可在主菜单中补充更多忽略前缀

## 分析流程

1. 读取文件
2. 基础特征统计
3. 指标计算
4. 问题判断
5. 推荐生成
6. 完成并刷新 UI

分析过程中：

- 进度窗口显示耗时、当前阶段和处理第 N/M 张。
- “取消分析”会取消本轮所有分析任务，清空本轮目标已产生的结果、失败状态、推荐修复和 cleanup candidates，但保留文件列表。
- 进度窗口关闭叉号等同于“取消分析”，不得直接关闭窗口留下半分析状态。
- 后台分析任务写回前必须校验当前分析轮次，已取消轮次的进度、结果和完成回调不得再更新 UI。

## 修复流程

1. 选择“修复当前”或“批量修复勾选”
2. 在修复弹窗中选择自动或手动方案；如需尝试 cleanup candidate，可额外勾选“强制修复不值得保留的图片”
3. 执行修复
4. 输出到目标目录或覆盖原文件
5. 保持 EXIF、ICC、DPI、XMP 等可保留信息
6. 降噪也走统一 repair planner / repair engine，而不是独立旧按钮

修复过程中进度窗口显示耗时、当前阶段和处理第 N/M 张；阶段文案面向用户解释为读取图片、生成修复方案、处理曝光/色彩/清晰度、保存结果等，不展示内部阈值或评分公式。

### 调试模式：修复后选择打开前后对比

- 主界面提供一个会话级复选框
- 默认关闭
- 开启后，仅在本轮修复成功后弹出调试打开窗口
- 窗口只列出本轮成功输出的图片，不混入历史结果
- 用户可勾选要打开的条目，然后批量打开原图和修复图
- 该功能不改变右侧预览区，不内置图片查看器，不改变修复输出逻辑

## 当前已知体验要求

- 右侧不显示大图片预览
- 条图区域尽量高，顶部说明尽量精简
- 弹窗底部按钮必须完整可见
- 条图信息尽量多显示，但不要以可读性为代价
## 1.1.3 Maintenance Addendum: Cleanup Candidate Panel

以下内容保留为 1.1.3 首次引入 cleanup candidate 面板时的历史说明；1.1.4 在其基础上继续扩展多选、主菜单复开和滚动完成详情。

- 主列表之外新增单独的“不适合保留候选”框体。
- 该框体只汇总 analyzer 产出的高风险 cleanup candidates，不会自动勾选任何图片。
- 当本轮分析刚结束且检测到新的 cleanup candidate 时，还会额外弹出一个本轮候选确认对话框，列出缩略图、文件名、主要原因、严重度和置信度。
- 支持：
  - 单个勾选
  - 批量切换所选候选
  - 全选
  - 取消全选
- 左侧主按钮用于执行安全清理：
  - 无勾选时按钮禁用
  - 有勾选后才启用
  - 执行前必须二次确认
- 安全清理顺序：
  - 优先尝试移入系统回收站
  - 若系统不支持，则移动到项目内 `_cleanup_candidates` 隔离目录
## 1.1.4 Maintenance Addendum: Multi-Select And Cleanup Workflow

- 主缩略图列表现在支持 `Ctrl` / `Shift` 多选。
- 多选时：
  - 右侧详情区显示汇总信息，而不是单张详情。
  - “修复勾选项”优先处理当前多选集合；如果没有多选，再回退到复选框勾选集合。
- 单选时仍保留原有的单图详情与预览行为。

### 不适合保留候选

- 左侧独立候选框体继续存在，并且与主缩略图列表联动：
  - 选择候选会自动定位主缩略图中的对应图片。
  - 从候选框体可以执行勾选当前、切换所选、全选、取消全选。
- 删除按钮位于左侧操作区，默认禁用；只有在至少勾选一张候选后才启用。
- 清理前必须二次确认。
- 安全清理顺序：
  - 优先尝试移入系统回收站
  - 如果系统不支持，则移动到项目内 `_cleanup_candidates` 隔离目录

### 批量修复完成窗口

- 批量修复完成后不再使用普通 `messagebox`。
- 新窗口分为两层：
  - 顶部摘要：已修复、已跳过、失败、输出目录、文件后缀、修复模式
  - 下方滚动详情：每张图的策略、方法、强度、跳过理由、no-op、回退、失败原因
- 窗口尺寸会受屏幕限制，避免超出可见区域。

### 批量修复内部策略

- UI 仍可显示“按检测结果自动推荐”，但内部已经变成 per-image adaptive 流程。
- 同一批中的不同图片会生成不同的 `repair_policy`、`selected ops` 与 `op_strengths`。
- cleanup candidate 默认不会进入修复；只有勾选强制尝试后，才允许进入候选评分与回退链路。
- 完成详情会区分正常修复、强制尝试修复后保存、强制尝试修复但回退，以及因仍不适合而跳过。

## 1.1.4 Maintenance Addendum: Scan Prefixes, Denoise And Mode Cleanup
- Settings/UI 补充：
  - 顶部“设置”菜单已改为统一“应用设置”入口。
  - 设置面板当前集中管理扫描忽略目录前缀、默认扫描模式和修复完成详情默认筛选。
  - “最近扫描摘要”窗口用于查看按前缀聚合后的跳过统计，以及每个被跳过目录的路径、命中前缀、跳过原因和层级位置。
  - 修复完成详情窗口现在支持按“已修复 / 已跳过 / 失败 / 强制尝试但未保存 / 强制尝试后保存 / 不适合保留相关 / 候选回退 / no-op”筛选。

- 目录扫描新增四选项范围选择，并统一写入扫描摘要。

- 独立“单图模式”入口已移除；单张图片仍通过主列表正常导入与分析。

## 1.1.4 Similar Images Workflow Addendum

- 相似组列表窗口的布局以列表 canvas 为唯一可扩展区域，底部“开始抉择 / 关闭”固定在窗口底部；窗口尺寸会按屏幕限制收敛，不超过常见屏幕可视区。
- 每个组卡片显示标题、勾选框、缩略图、相似度说明和完整文件名摘要；组内图片较多时最多预览前 6 张并显示剩余数量，避免撑出过高卡片。
- 筛选项区分“高相似 / 中等相似 / 低置信候选 / 组内数量较多 / 可能连拍/同组拍摄”。
- 分析完成后，应用会在后台检测本轮图片的相似组，并在 Console 与分析摘要中显示相似组数量。
- 如果存在相似组，先保留现有 cleanup candidate 复核流程，再显示相似组列表，避免多个重要弹窗同时遮挡。
- 相似组列表窗口支持滚动查看、按“全部相似组 / 高相似度 / 中等相似度 / 组内数量较多 / 可能连拍”筛选，并支持多选组。
- 用户点击“开始抉择”后，只进入已勾选组；未勾选组保留在列表中，不自动处理。
- 组内对比窗口每次展示一个组：2-4 张使用自适应网格，更多图片按最多 4 张分页切换；窗口过小时提示用户放大窗口或跳过本组。
- 每张图显示预览、文件名、风险值、场景、人像判断、问题摘要、推荐修复和 cleanup candidate 标记；“删除此图”必须二次确认。
- 相似图删除复用全局 `safe_cleanup_paths()`：优先回收站，失败后进入 `_cleanup_candidates`，再失败则弹窗显示原因。
- 删除成功后刷新相似组、主缩略图列表、cleanup candidate 列表和文件存在状态；未删除图片的分析结果、修复建议和扫描摘要不变。
