# 1.1.4

日期：`2026-05-06`

## 版本概览

### 场景感知分析与分析器模块化

- 分析器已从单文件实现拆分为 `analysis/` 包，并保留 `analyzer.py` 兼容入口。
- 新分析结果会输出 `scene_type`、`portrait_type`、`exposure_type`、`highlight_recovery_type`、`color_type` 等场景字段，供修复规划、批量结果解释与回归维护使用。

### 人像误判治理

- 人像识别改为 raw candidate 与 validated real face 双层结构。
- 新逻辑能区分真实正面/近正面人脸、背身或侧背身人物、画作/海报中的脸，以及纹理类误检。
- 低置信度候选不再进入 portrait policy、真人虚焦建议删除和多人像评分。

### 通用“不适合保留”机制

- cleanup candidate 从“人像虚焦特例”升级为通用机制。
- 本轮已接入真实正面人像严重虚焦、严重全图糊片与极端不可恢复曝光问题。
- UI 支持主菜单再次打开候选列表、状态刷新、批量勾选与安全移入回收站/隔离目录。

### 自动修复策略升级

- 批量修复已改为单图单策略、单图单力度。
- 每张图先生成独立 `RepairPlan`，再执行候选评分、风险限制、回退与 no-op 决策。
- 高反差窗景、剪影、低调氛围、高光不可恢复天空/白墙、自然高饱和场景都加入了额外保护。

### 批量体验与线程安全

- 目录扫描、批量分析、批量修复都使用受控线程池执行。
- Tk UI 更新统一回到主线程。
- 输出路径生成已加锁，减少同名输出在并发场景下的冲突。
- 批量修复完成提示改为可滚动自定义对话框，完整显示跳过、回退与失败原因。

### 目录扫描忽略前缀与四选项

- 主菜单新增“设置 -> 扫描忽略目录前缀”，使用轻量 JSON 持久化忽略前缀。
- 默认至少保留 `_repair`，因此 `_repair`、`_repair_old`、`_repair_2026`、`_repaired` 等目录都会在任意层级被整体跳过。
- 当目录包含子目录时，扫描前会弹出四选项对话框：
  - 扫描全部，包含子目录
  - 只扫描当前目录
  - 扫描所有子目录
  - 取消扫描
- 扫描摘要会输出扫描模式、跳过目录数量与导入图片数量；拖拽文件夹导入也统一遵守这套规则。
- 四选项对话框现在使用受屏幕高度限制的滚动内容区，底部“取消扫描”固定可达，避免小屏或系统缩放时被遮住。

### 分析取消与进度体验

- 分析进度窗口新增“取消分析”按钮；窗口关闭叉号等同于取消分析，统一走同一套清理逻辑。
- 取消会设置本轮分析取消标记，清空本轮目标已经写入的 `results`、`errors`、推荐修复信息和 cleanup candidate 选择状态，但保留文件列表与勾选状态。
- 后台分析任务写回前必须匹配当前分析轮次；已取消轮次的进度、结果和完成回调会被丢弃，避免半分析状态覆盖 UI。
- 取消后主界面回到“已添加文件但尚未分析”的状态，可以重新点击“分析全部”或“分析选中”。
- 分析和修复进度窗口新增耗时显示，并展示“读取图片、分析曝光/色彩/清晰度、生成建议、处理第 N/M 张、保存结果”等用户可理解的阶段信息，不暴露内部阈值或评分公式。

### cleanup candidate 强制尝试修复

- 修复准备窗口新增“强制修复不值得保留的图片”，默认不勾选。
- 默认状态下，被标记为 discard candidate / cleanup candidate 的图片不会进入修复尝试。
- 勾选后，这些图片可以进入统一 `repair_planner.py` + `repair_engine.py` 链路，但仍按单图分析结果选择方法与力度，不会统一套用同一算法。
- 强制尝试后仍需通过单图候选评分、安全检查和附加可保留性判断；失败时继续回退或跳过保存。
- 批量完成详情会区分正常修复、强制尝试修复后保存、强制尝试修复但回退，以及因仍不适合而跳过。

### 去噪并入统一分析与修复链

- 分析结果新增 `noise_score`、`noise_level`、`denoise_profile`、`denoise_recommended`，用于判断是否应该把降噪纳入修复方案。
- 旧的“去噪当前”按钮已移除；降噪统一通过 repair planner / repair engine 执行。
- 降噪策略现已覆盖：
  - 人像：优先保护脸部与皮肤纹理，避免塑料感
  - 夜景 / 暗部 / 高 ISO：优先抑制阴影颗粒
  - 纯净天空 / 大面积平滑背景：允许更明显的低纹理降噪
  - 建筑 / 文字 / 强纹理：严格保护边缘与结构细节
- 降噪结果也会进入候选评分和修复后安全检查，若出现糊化或细节损失，会降分、回退或跳过输出。

### 单图模式移除

- 主界面已移除独立“单图模式”入口。
- `single_image_window.py` 已从项目中移除，相关入口和依赖链一并断开。
- 选择单张图片进入主列表分析的能力保持不变；移除的是额外的独立窗口模式，而不是单图导入能力。

## 相似图片自动检测与复核删除

- 本轮继续归入 1.1.4，不提升版本号。
- 分析批次完成后会额外生成批次级 `similar_groups`，只记录相似组路径、相似度、等级、依据和可能连拍标记，不改写单张 `AnalysisResult` 中已有的 `issues`、`scene_type`、人像字段、修复建议、修复力度或 cleanup candidate。
- 相似检测使用后台线程，在单图分析线程池完成后继续执行；Tk 更新、弹窗和列表刷新仍回到主线程。
- 检测算法只读取缩略图和摘要特征：32px 颜色/亮度摘要、aHash、dHash、16px 灰度结构向量、尺寸比例、EXIF 拍摄时间。100 张以内默认只比较轻量特征，不做全尺寸两两比较；更大批次会使用哈希、尺寸和时间分桶减少候选对。
- Console 和分析完成摘要会输出相似组数量；每个相似组记录组内数量、相似度和主要依据。
- 新增相似组列表窗口，支持滚动、多选和按“全部相似组 / 高相似度 / 中等相似度 / 组内数量较多 / 可能连拍”筛选。用户点击“开始抉择”后只处理勾选组。
- 新增组内对比窗口，2 张、3-4 张走自适应网格，更多图片按最多 4 张分页轮播；每张图显示预览、文件名、风险值、场景、人像/非人像、问题摘要、修复建议和 cleanup candidate 标记。
- 相似图删除复用 `safe_cleanup_paths()`：先尝试系统回收站，失败后进入项目 `_cleanup_candidates` 安全隔离目录；若隔离也失败，弹窗提示原因，不执行永久删除。
- 删除后会更新相似组、主缩略图列表、cleanup candidate 列表和文件存在状态；未删除图片的分析结果与修复建议保持不变。
## Similar Image Detection And Review Fix

- 本轮仍写入 1.1.4，只修复相似图片检测和相似图复核窗口。
- 检测算法新增多尺度中心裁切、旋转鲁棒 aHash/dHash、场景/中心主体摘要、边缘摘要、文件编号连续性和可靠 EXIF/文件时间辅助；横竖变化、裁切变化、主体大小变化明显的同场景同主体图片可进入“中等相似”或“低置信候选”。
- 相似组分级扩展为 `high`、`medium`、`low`：高相似用于近重复和轻微位移；中等相似用于同主体同场景但构图差异明显；低置信候选只供人工复核。
- 相似组列表窗口修复滚动布局：列表 canvas 所在行独占可扩展空间，底部“开始抉择 / 关闭”固定在底部；组卡片按内容自然高度排列，不再出现半组内容被截断但底部大片空白。
- UI 筛选新增“低置信候选”，并保留高相似、中等相似、组内数量较多、可能连拍/同组拍摄。
