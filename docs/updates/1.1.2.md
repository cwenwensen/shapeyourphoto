# 1.1.2

日期：`2026-05-02`

## 本轮改进

### 饱和度相关算法调整

- 重新审视“饱和度偏高”的判定方式，避免把自然绿植、花丛、阴影里的高纯度颜色直接误判为需要整图去色。
- 将判定重点从粗暴的全局饱和统计，调整为更关注中间调和亮部的异常高饱和区域。
- 对绿植占比高、亮部不刺眼、问题主要集中在阴影颜色的场景增加减敏保护。
- 修复策略从“整图统一降饱和”改为“定向压制中高亮区域的极端颜色”，减少修复后整图发灰的问题。

### 修复输出方向一致性

- 修复竖图在连续修复后发生 90 度旋转的问题。
- 修复时对像素数据做 `exif_transpose` 后，输出 EXIF 方向统一归一为 `Orientation = 1`，确保原图显示方向与修复结果一致。

### 文档体系补充

- 正式建立 `docs/technical/` 与 `docs/updates/` 两个子目录。
- `technical/` 用于长期技术维护文档。
- `updates/` 用于每一轮版本改进文档，按版本号单独归档。
