# Update Docs

`docs/updates/` 用于存放每次版本更新对应的改进记录文档。

规则：

1. 后续接手的 agent 不得删除 `docs/updates/`。
2. 每一轮正式改动应新增一份以版本号命名的文档，例如 `1.1.2.md`。
3. 更新文档应记录该版本改了什么、为什么改、涉及哪些关键文件、如何验证。
4. 旧版本更新文档不得删除；如需补充，应追加修订。
