# 1.1.3

日期：`2026-05-05`

## 版本概览

### 人像感知误修抑制

- 在不引入 `opencv` 等新重依赖的前提下，为分析链路加入轻量人像候选检测。
- `AnalysisResult` 增加人脸数量、候选框、脸部亮度、主体亮度、背景亮度、人像曝光状态和诊断标签。
- 当检测到人像且主体曝光正常时，不再仅因全图偏暗直接判定为欠曝。
- 对闪光灯人像、暗背景毕业照、黑色学士服场景，诊断会更偏向“主体正常、背景偏暗但可作为氛围”。

### Portrait-Aware 局部增强

- 在现有 face box 基础上继续估计 `face_region`、`subject_region`、`background_region`、`highlight_region`。
- 新增 `portrait_scene_type`、`face_exposure_status`、`subject_exposure_status`、`background_exposure_status`、`highlight_clipping_ratio`、`subject_background_separation`、`portrait_repair_policy`。
- 高亮背景人像会被识别为 `high_key_portrait` 或 `backlit_portrait`，不再默认规划强全局压高光。
- 对高调背景人像优先采用局部主体增强和背景保护，而不是把白墙、浅色建筑压成灰白。

### 自动修复更保守且更分区域

- 自动修复方案不再对 `portrait_subject_ok` 场景优先规划强全局提亮。
- 暗背景人像不再优先使用 `auto_tone`，并收紧 `lift_shadows` 强度。
- 新增局部方法：
  - `portrait_local_face_enhance`
  - `portrait_subject_midcontrast`
  - `portrait_dark_clothing_detail`
  - `protect_high_key_background`
- 局部处理使用粗 mask + 羽化 + alpha blend，强度保持保守。

### 候选评分与回退

- 人像场景修复不再直接保存单一结果，而是先生成保守强度候选。
- 候选会比较脸部亮度、脸部饱和度、主体背景分离、深色服装亮度、高调背景亮度和全图饱和度。
- 如果候选未优于原图，会自动回退为 no-op，并在 Console 输出“不建议自动修复：原图人像主体已较正常。”
- 对亮背景过曝人像，若全局压高光会把白墙或浅色建筑压灰，则会改为高调背景保护加局部主体增强，而不是直接跳过。
- 所有跳过场景都会生成显式跳过理由，并显示在修复完成提示框中。

### 修复后安全提示与 UI 细节

- 修复后新增轻量安全检查，当前只提示、不阻断保存。
- 可提示人脸亮度提升过多、黑色区域明显变灰、整体饱和度下降、背景被过度抬亮、高调背景被压灰。
- 右侧诊断区和 Console 增加人像场景类型、修复策略、候选接受/回退结果与保护提示。
- Console 刷新统一回到主线程，减少后台任务直接触碰 Tk UI 的风险。

## 验证要点

- 闪光灯人像 / 暗背景人像不再被直接判定为欠曝并强行全局提亮。
- 高亮白墙 / 浅色建筑人像不再被默认整体压暗，优先走局部主体增强和背景保护。
- 普通欠曝样图仍会继续触发欠曝诊断与提亮修复。
- 普通过曝非人像仍会继续触发全局压高光。
- 候选评分能在无收益时自动 no-op 并跳过输出。
- 批量分析仍可在后台线程执行，UI 只在主线程更新。
- 修复输出路径、EXIF / ICC / DPI / XMP 保留逻辑保持不变。
- `start.bat` 未改动，仍保持快速启动入口。

## 后续维护追加

- 人脸检测补充分层机制：
  - `raw_face_candidates` 允许偏宽松
  - `validated_face_boxes` 只保留通过复核的更可靠结果
  - `face_candidates` 记录每个候选的拒绝原因
- 新增 `portrait_out_of_focus`，用于识别人像主体严重未对焦。
- 新增通用 `cleanup_candidates` 结构和 UI 候选框体，用于统一承载“不适合保留”图片，而不只服务于人像虚焦。
- 当分析结束且本轮命中 cleanup candidate 时，会弹出一个单独的本轮候选确认框，默认不勾选任何图片，用户可在对话框内批量勾选并进入安全删除确认流程。
- 安全清理优先进入系统回收站，失败时自动回退到项目 `_cleanup_candidates` 隔离目录。
- 修复少量分析结果文案显示为乱码的问题，并增加基于问题代码的文案兜底回退，确保问题标签、详情和建议始终可读。
